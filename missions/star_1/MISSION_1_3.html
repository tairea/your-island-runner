Mission 1.3: Add Danger

What separates a playground from a game? Consequences. Right now, your jumping course is just practice - miss a jump and nothing happens. But in a real game, missing a jump should matter. In this mission, you are going to write your first lines of code and create something that makes players think twice before they leap.

You are about to become a programmer. Not just someone who places blocks, but someone who makes those blocks do things. By the end of this mission, your game will have a deadly kill brick that resets players who touch it. One small script, one huge leap forward.

Programming is Power: Every video game you have ever played runs on code. Code makes enemies chase you, coins disappear when collected, and checkpoints save your progress. Today, you join the ranks of people who write that code.

Your Mission

Learning Objective: Create a kill brick using a simple script that sends players back to the start when touched.

What You Will Create

  • A distinctive red "danger zone" part below your platforms
  • Your first Lua script
  • Code that detects when a player touches the danger zone
  • A consequence: touching the danger resets the player

Step-by-Step Instructions

Step 1: Create the Danger Zone

First, we need a part that represents danger. Let us create a big red zone below your jumping course.

  1. Go to the Model tab and insert a new Part (Block)
  2. Use the Scale tool to make it large - about 50x1x50 studs
  3. Use the Move tool to position it directly below your platforms, about 10 studs down

Now let us make it look dangerous:

  1. With the part selected, look at the Properties panel on the right side of the screen
  2. If you do not see Properties, go to View tab and click Properties
  3. Find the BrickColor property and click on it
  4. Select Bright red or Really red
  5. Find the Name property at the top and change it from "Part" to KillBrick

Naming Matters: Always give your parts descriptive names! When you have hundreds of parts, you will thank yourself for naming them clearly. "KillBrick" is much easier to find than "Part" or "Part47".

Step 2: Add a Script to Your Kill Brick

Parts just sit there until you add code. Let us give your kill brick a brain!

  1. In the Explorer panel (usually on the right), find your KillBrick
  2. Right-click on KillBrick
  3. Hover over Insert Object
  4. Click Script

A new Script appears inside your KillBrick, and a code editor opens! You will see some default text:

print("Hello world!")

This default code just prints a message. We are going to replace it with our kill brick code.

Step 3: Write Your First Real Code

Delete everything in the script and type this code exactly:

local killBrick = script.Parent

local function onTouched(otherPart)
    local humanoid = otherPart.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = 0
    end
end

killBrick.Touched:Connect(onTouched)

Type Carefully! Code must be typed exactly right. Every capital letter, every period, every parenthesis matters. If something does not work, check your spelling and punctuation first!

What Does This Code Do?

Let us break it down line by line:

local killBrick = script.Parent

This stores a reference to the part containing this script (your KillBrick).

local function onTouched(otherPart)

This creates a function named "onTouched" that runs whenever something touches our brick. The "otherPart" is whatever touched us.

local humanoid = otherPart.Parent:FindFirstChild("Humanoid")

This checks if the thing that touched us is part of a player character. Players have a "Humanoid" component that controls their health and movement.

if humanoid then

This checks: "Did we actually find a Humanoid?" We only want to kill players, not random parts.

humanoid.Health = 0

This sets the player's health to zero, which eliminates them!

killBrick.Touched:Connect(onTouched)

This connects our function to the brick's Touched event. Now, whenever anything touches the brick, our onTouched function runs.

Step 4: Test Your Kill Brick

Time to see if your code works!

  1. Click Play
  2. Walk off the edge of the baseplate and fall onto your red kill brick
  3. Watch what happens!

If everything worked, you should see your character disappear and respawn at the starting position. Congratulations - your first script works!

Try It!

Jump off from different heights. Try landing on the edge of the kill brick. Does it work every time? Try missing a platform jump and falling - does the kill brick catch you?

Step 5: Troubleshooting (If It Did Not Work)

If your kill brick is not working, here are common problems and fixes:

Nothing happens when I touch it

  • Check that the Script is inside the KillBrick in the Explorer
  • Make sure you typed the code exactly right
  • Check for the red underline in the script editor - this shows errors

Red error messages in the Output

  • Open the View tab and click Output
  • Read the error message - it tells you what line has a problem
  • Common issues: missing parentheses, typos in "Humanoid", wrong capitalization

I fall through the kill brick

  • Make sure the KillBrick's CanCollide property is set to true
  • Check that the brick is not Anchored to false (it should be Anchored = true)

The Output Window is Your Friend: Whenever something is not working, check the Output window (View > Output). Error messages appear there in red and tell you exactly what went wrong and on which line.

Step 6: Fine-Tune the Position

Make sure your kill brick is positioned well:

  • It should be below all your platforms so missing any jump is deadly
  • It should be large enough that players cannot fall past it
  • Consider adding walls around the edges so players cannot escape sideways

Play your course a few times. The danger is real now - every jump matters!

Code Reference: Kill Brick Script

local killBrick = script.Parent

local function onTouched(otherPart)
    local humanoid = otherPart.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = 0
    end
end

killBrick.Touched:Connect(onTouched)

Key Concepts:

  • script.Parent - References the part containing this script
  • Touched - An event that fires when something touches the part
  • :Connect() - Connects a function to an event
  • Humanoid - The component that makes a character a "player"
  • Health = 0 - Setting health to zero defeats the player
