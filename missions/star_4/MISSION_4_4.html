Mission Objective

Code kill bricks to return players to checkpoints

Understanding Death in Roblox

In Roblox, every player character has a Humanoid object that controls their health. Here's how it works:

  • Humanoid.Health - The character's current health (default is 100)
  • Humanoid.MaxHealth - The maximum health (also default 100)
  • When Health reaches 0, the character dies
  • When a character dies, Roblox automatically respawns them at their RespawnLocation

So to create a kill brick, all we need to do is find the player's Humanoid and set its Health to 0. Simple!

Step 1: Create Your First Kill Brick

Let's create a dangerous lava pit!

  1. In the Workspace, create a new Part
  2. Rename it to KillBrick
  3. Make it big enough to be a challenge - try size (20, 1, 20)
  4. Change its color to Bright red or Really red
  5. Change its Material to Neon to make it look dangerous
  6. Position it somewhere players might fall - like between two platforms or at the bottom of a jump
  7. Make sure CanCollide is checked so players can actually touch it
  8. Make sure Anchored is checked so it doesn't fall

Design Tip: Make Danger Obvious

Good game design means players should KNOW something is dangerous just by looking at it. Use bright red, orange, or black colors. Add a Neon material to make it glow. Maybe even add fire particles! Players should never die and think "How was I supposed to know that was dangerous?"

Step 2: Write the Kill Script

Right-click your KillBrick and insert a Script. Rename it to KillScript.

Here's the complete code:

local killBrick = script.Parent

killBrick.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")

    if humanoid then
        humanoid.Health = 0
    end
end)

Wait... that's it? That's ALL the code we need? Yes! Let's break it down:

Line by Line Explanation

Line 1: Reference the Part

local killBrick = script.Parent

Standard setup - create a reference to the part containing this script.

Lines 3-8: The Touched Event

This should look very familiar! When something touches the kill brick:

  • Line 4: Look for a Humanoid in whatever touched the brick
  • Line 6: If we found a Humanoid, check if it exists
  • Line 7: Set its Health to 0 (which kills it instantly)

That's it! When Health hits 0, Roblox automatically handles the death animation and respawning the player at their last checkpoint.

Step 3: Test Your Kill Brick

  1. Make sure you have at least one checkpoint in your game (from Mission 4.3)
  2. Click Play
  3. Touch a checkpoint first (so you have a respawn point)
  4. Walk onto the kill brick
  5. Your character should die instantly!
  6. You should respawn at the checkpoint you touched

SUCCESS! You just created your first deadly obstacle!

Make It Better: Add a Warning Message

Let's add a print message so we can track when players die (useful for testing!):

local killBrick = script.Parent

killBrick.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")

    if humanoid then
        local player = game.Players:GetPlayerFromCharacter(hit.Parent)
        if player then
            print(player.Name .. " died!")
        end
        humanoid.Health = 0
    end
end)

Step 4: Create Different Types of Kill Zones

One kill brick is good, but let's make your game more interesting with different types of hazards!

Lava Pit

  • Color: Bright red or Really red
  • Material: Neon
  • Add a Fire object inside the part (Insert Object > Fire)
  • Make it wide and flat, like a pool of lava

Spike Trap

  • Color: Dark stone grey or Black
  • Material: Slate or Concrete
  • Make it smaller and thinner - like actual spikes
  • You can create multiple thin parts grouped together to look like spikes

Poison Water

  • Color: Bright green or Lime green
  • Material: Glass or Neon
  • Transparency: 0.3 (so you can see through it a bit)
  • Make it look like a puddle or pool

The Void (For Falling Off)

  • Create a HUGE kill brick far below your island
  • Size: (500, 1, 500) or bigger
  • Color: Black
  • Transparency: 1 (completely invisible)
  • This catches players who fall off the island!

Duplicate Your Kill Bricks

Just like with checkpoints, you can duplicate kill bricks (select the brick AND its script, then press Ctrl+D). Place dangerous obstacles throughout your island:

  • Between difficult jumps
  • Below narrow paths
  • As moving obstacles (we'll learn moving parts later!)
  • Around the edges to catch players who fall off

Step 5: Add Advanced Features (Optional)

Let's make kill bricks even more interesting!

Option 1: Damage Over Time (DOT) Instead of Instant Death

What if instead of instant death, the brick slowly drains health?

local damageBrick = script.Parent
local damageAmount = 10
local debounce = {}

damageBrick.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")

    if humanoid and not debounce[humanoid] then
        debounce[humanoid] = true
        humanoid.Health = humanoid.Health - damageAmount
        print("Player took " .. damageAmount .. " damage!")

        wait(1)
        debounce[humanoid] = nil
    end
end)

How it works:

  • Instead of setting Health to 0, we subtract 10 each time
  • Debounce ensures damage only happens once per second
  • Players can walk across it if they're quick, but lingering will kill them!

Option 2: Play a Sound on Death

Add audio feedback to make deaths more impactful!

  1. Insert a Sound object into your kill brick
  2. Set the SoundId to a death sound (search the toolbox for "death sound" or "explosion")
  3. Add this line before humanoid.Health = 0:
script.Parent.Sound:Play()

Option 3: Explosion Effect

Make deaths more dramatic with an explosion!

local killBrick = script.Parent

killBrick.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")

    if humanoid then
        -- Create explosion at player position
        local explosion = Instance.new("Explosion")
        explosion.Position = hit.Position
        explosion.BlastRadius = 5
        explosion.BlastPressure = 0  -- Don't push other parts around
        explosion.Parent = workspace

        humanoid.Health = 0
    end
end)

Step 6: Balance Your Game

Now that you have checkpoints AND kill bricks, you need to balance your game. This is game design!

Game Balance Principles

  • Fair Challenge: Make sure checkpoints come BEFORE hard sections, not after
  • Clear Danger: Players should always see hazards before touching them
  • Progressive Difficulty: Start easy, get harder as players progress
  • Test, Test, Test: Play your own game! If YOU get frustrated, players will too

Walk through your island as a player would. Ask yourself:

  • Is this challenge fair?
  • Can I see the danger before I hit it?
  • Is there a checkpoint close enough that dying isn't too frustrating?
  • Is this section too easy? Too hard?

Common Problems and Solutions

Problem: Nothing happens when I touch the kill brick

Solutions:

  • Make sure CanCollide is checked (otherwise you'll fall through it)
  • Verify the script is inside the kill brick part
  • Check for errors in the Output window
  • Make sure you're testing with a character, not just moving the camera

Problem: I die but respawn at the starting spawn, not my checkpoint

Solution: You need to touch a checkpoint FIRST to set your RespawnLocation. Go back to Mission 4.3 and make sure your checkpoints are working correctly.

Problem: Other parts that touch the kill brick disappear

This is normal! The kill brick will destroy anything with a Humanoid. If you don't want this, add a check to only kill players specifically (check if a Player object exists).

Problem: My character dies repeatedly even after leaving the kill brick

Solution: The Touched event might be firing multiple times. Add debounce (like we did in checkpoints) to prevent repeated deaths.

The Psychology of Death in Games

Here's something cool to think about: death in games isn't bad! Death creates:

  • Stakes: If you can't fail, success doesn't feel good
  • Learning: Players learn what NOT to do
  • Tension: Danger makes the game exciting
  • Accomplishment: Beating a hard section feels AMAZING because you could have failed

The key is making death feel FAIR. If players die and think "That was my fault, I can do better next time," that's good game design. If they think "That was cheap!" or "How was I supposed to avoid that?" - that's bad design.

Pro Game Designer Tip

Watch other people play your game (siblings, friends, classmates). Don't help them! Just watch where they die and listen to their reactions. If they die in the same spot over and over and get frustrated, you might need to add a checkpoint or remove some hazards. If they die but immediately want to try again, you nailed the difficulty!
