Your First Line of Code

Everything you've built so far - your island, your paths, your landmarks - they're all just sitting there, waiting. What if your game could actually do something? What if it could react, respond, and come alive?

Welcome to the world of coding. You're about to write your first script, and trust me - this is where the real magic begins.

Mission Objective

Create a Script, write print(), and find it in the Output window

What is a Script?

Think of a script like a recipe. Just like a recipe tells you step-by-step how to make cookies, a script tells Roblox step-by-step what to do in your game. Want a door to open when a player touches it? You need a script. Want to keep track of points? You need a script. Want your game to do literally anything interactive? You guessed it - scripts!

In Roblox, we write scripts using a programming language called Lua. Don't worry - Lua is one of the friendliest coding languages out there, designed specifically for games.

Step 1: Create Your First Script

  1. In the Explorer panel, find ServerScriptService
  2. Right-click on ServerScriptService
  3. Select Insert Object > Script
  4. A new script appears! Rename it to HelloWorld

Why ServerScriptService?

This is where we put scripts that run on the game server (the computer hosting your game). These scripts control the main game logic that everyone playing needs to follow. Later, you'll learn about other places to put scripts, but for now, ServerScriptService is your home base.

Step 2: Your First Line of Code

Double-click your HelloWorld script to open it. You'll see it already has some code in it - something that says print("Hello world!"). That's Roblox being helpful!

Let's understand what this code means:

print("Hello world!")

Breaking It Down Line by Line

  • print - This is a function (a pre-made tool) that displays messages
  • ( - The opening parenthesis starts the function's instructions
  • "Hello world!" - This is the message we want to display. The quotation marks tell Lua "this is text, not code"
  • ) - The closing parenthesis ends the function's instructions

Think of it like this: You're asking Lua to print something, and inside the parentheses, you tell it exactly what to print.

Step 3: Customize Your Message

Let's make this script your own! Change the message to something personal:

print("Welcome to my island runner!")
print("Get ready for an adventure!")
print("Created by [Your Name]")

Try It: Multiple Messages

Notice how we have three different print statements? Each one is its own line of code. Try adding your own messages! You can print anything you want - song lyrics, jokes, facts about yourself. Just remember to keep the text inside quotation marks.

Step 4: Find the Output Window

Now comes the exciting part - seeing your code work!

  1. At the top of Roblox Studio, click the View tab
  2. Click Output to open the Output window (if it's not already visible)
  3. The Output window usually appears at the bottom of your screen

The Output Window is Your Best Friend

Seriously. Every coder - beginner or expert - relies on the Output window. It's where you see messages from your code, and more importantly, it's where you see error messages that help you fix problems. Get comfortable with this window!

Step 5: Run Your Code

Time to see your code in action!

  1. Click the Play button at the top of Roblox Studio (the blue arrow)
  2. Look at the Output window
  3. You should see your messages appear!

It should look something like this:

Welcome to my island runner!
Get ready for an adventure!
Created by Alex

CONGRATULATIONS! You just wrote and ran your first code! That feeling you have right now? That's the feeling every programmer gets when their code works. Hold onto that.

Understanding What Just Happened

When you clicked Play, here's what happened behind the scenes:

  1. Roblox started running your game
  2. It looked in ServerScriptService for scripts to run
  3. It found your HelloWorld script
  4. It read each line of code from top to bottom
  5. When it saw print(), it sent those messages to the Output window

Experiment Time!

Try these experiments to learn more:

  • Change the order: Move your print statements around. What happens?
  • Break it (on purpose): Delete one of the quotation marks and run your code. What error message do you get? This is how you learn to debug!
  • Print numbers: Try print(42) without quotes. Numbers don't need quotes!
  • Do math: Try print(10 + 5). Lua can do calculations!

Common Mistakes (And How to Fix Them)

Problem: I don't see any messages in the Output window

Solutions:

  • Make sure you clicked the Play button
  • Check that the Output window is actually open (View > Output)
  • Verify your script is in ServerScriptService, not somewhere else

Problem: I see a red error message

Solutions:

  • Check that every opening quotation mark " has a closing quotation mark "
  • Make sure every opening parenthesis ( has a closing parenthesis )
  • Verify you spelled "print" correctly (lowercase, no spaces)
  • Read the error message - Roblox tries to tell you what's wrong!

Problem: My script disappeared!

Solution: It's probably still there, just in a different folder. Use the search bar at the top of the Explorer to find it by name.

Why This Matters

I know - printing messages doesn't seem like much. But you just learned something HUGE:

  • How to create and organize scripts
  • How to write code that Lua understands
  • How to see if your code is working
  • How to use the Output window for debugging

Every complex game mechanic - from checkpoint systems to boss battles - starts with these basics. You're building your foundation.

Pro Coder Tip

Professional game developers use print() ALL THE TIME. When something isn't working, they add print statements to figure out what's happening. It's like leaving yourself notes to see what your code is doing. Never think print() is "too basic" - it's one of the most powerful debugging tools you have!
