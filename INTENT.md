# 1. Audience (Who)
- **Age/Level**: Intermediate and high school students
- **Prior Knowledge**: Complete beginners - no prior Roblox Studio or coding experience
- **Context**: Classroom setting, weekly lessons over a school term
- **Motivation**: Create a real, playable game they can share with friends

---

# 2. Topic/Domain (What)
- **Subject**: Roblox Studio Game Development - Obstacle Runner
- **Scope**:
  - Roblox Studio interface and navigation
  - Building with parts (obstacles, platforms, terrain)
  - Basic Lua scripting (checkpoints, respawns, win condition)
  - Using satellite imagery as reference for world design
  - Playtesting and debugging
  - Publishing to Roblox
- **Focus Areas**: Environment building, core game mechanics, publishing workflow
- **Out of Scope**: Advanced scripting/OOP, custom animations/models, monetization, complex UI systems

---

# 3. Unique Ideas
- **Approach**: Project-based learning - building one cohesive game throughout the course
- **Theme**: "Your Island Runner" - each student creates an obstacle course based on the shape and landmarks of their local island, city, or town using satellite imagery (Google Maps) as reference
- **Differentiation**: Personalized geography makes each student's game unique and meaningful, increasing engagement and making sharing with friends more exciting

---

# 4. Outcomes
- **Knowledge**: Understand how Roblox games are structured (parts, scripts, game flow, publishing)
- **Skills**: Navigate Roblox Studio, build environments, write basic Lua scripts, playtest and debug, publish games
- **Artifacts**: A published Roblox obstacle runner game based on their local area that friends can actually play
- **Certification**: Published game link serves as portfolio piece and proof of completion

---

# 5. Timing
- **Total Duration**: 20-25 hours
- **Session Length**: 2-3 hours per week
- **Pacing**: Scheduled weekly class over 8-10 weeks (school term)
- **Depth**: Solid working knowledge - enough to build, publish, and extend their own games

---

# 6. Assessment/Mastery
- **Formative**: Required teacher check-in submissions at key milestones (world shape complete, first obstacle working, checkpoint system done, etc.)
- **Summative**: Final published game playable by friends
- **Proof of Mastery**: Completion of all milestone check-ins + published Roblox game
- **Progression Rules**: Must complete each check-in submission before advancing to next section
